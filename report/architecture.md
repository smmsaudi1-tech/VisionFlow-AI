# بنية وهيكلية مشروع VidAI Studio

مشروع **VidAI Studio** هو منصة سحابية لتوليد وتعديل مقاطع الفيديو تلقائياً باستخدام الذكاء الاصطناعي. يتكون المشروع من شقين رئيسيين: واجهة مستخدم (Frontend) مبنية بـ Next.js، وخلفية برمجية (Backend) مبنية بـ FastAPI.

---

## 1. الهيكل العام للمشروع (Project Structure)

```text
يوتيوب/
├── backend/               # الخلفية البرمجية (FastAPI)
│   ├── models/            # نماذج قواعد البيانات و Pydantic schemas
│   ├── routers/           # مسارات الـ API (analyze, render)
│   ├── services/          # منطق الأعمال (FFmpeg, TTS, Whisper, YouTube)
│   ├── main.py            # نقطة البداية وتشغيل السيرفر
│   └── state.py           # إدارة الحالات المخزنة في الذاكرة المؤقتة (TTL Store)
│
├── frontend/              # واجهة المستخدم (Next.js)
│   ├── app/               # صفحات التطبيق (الصفحة الرئيسية، اللوحة، الاستوديو، التحليل)
│   ├── components/        # المكونات المشتركة (HeroInput, etc.)
│   ├── lib/               # استدعاءات الـ API والمكتبات المساعدة
│   └── package.json       # إعدادات الواجهة والمكتبات المستخدمة
│
├── docker-compose.yml     # تشغيل المشروع بالكامل بـ Containers
└── wrangler.toml          # إعدادات الرفع والاستضافة على Cloudflare
```

## 2. بنية الخلفية البرمجية (Backend Architecture)

تم بناء الخلفية البرمجية باستخدام **FastAPI** لتوفير أداء سريع للغاية ومعالجة متزامنة (Asynchronous) لعمليات الرندر والتحليل المعقدة.

### أ. إدارة الحالات (State Management)
* **الملف:** [state.py](file:///c:/Users/youse/OneDrive/Desktop/يوتيوب/backend/state.py)
* **الوصف:** يحتوي على كلاس `TTLStore` الذي يقوم بتخزين بيانات المشاريع والرندر في الذاكرة المؤقتة لـ FastAPI تلقائياً مع تحديد عمر (TTL) لكل سجل (ساعتان) لتجنب حدوث تسريب في الذاكرة (Memory Leaks) أثناء معالجة الفيديوهات الثقيلة.

### ب. المسارات (Routers)
1. **[analyze.py](file:///c:/Users/youse/OneDrive/Desktop/يوتيوب/backend/routers/analyze.py):**
   * يتعامل مع طلبات تحليل الفيديوهات/النصوص.
   * يستخدم الذكاء الاصطناعي لتوليد المقاطع النصية (Scenes) واقتراح الفيديوهات المناسبة.
2. **[render.py](file:///c:/Users/youse/OneDrive/Desktop/يوتيوب/backend/routers/render.py):**
   * يتعامل مع طلبات رندر الفيديوهات النهائية (توليد الصوت، دمج المقاطع، إضافة الكابشن).
   * يقوم بعملية الـ Polling لمتابعة حالة الرندر خطوة بخطوة.

### ج. الخدمات (Services)
* **FFmpeg Service (`ffmpeg_service.py`):** دمج مقاطع الفيديو، قصها، كتم صوتها، ودمج الصوت المتولد (Audio track) والكابشنز عليها.
* **TTS Service (`tts_service.py`):** توليد التعليق الصوتي للمشاهد باستخدام نماذج تحويل النص إلى كلام.
* **Whisper Service (`whisper_service.py`):** معالجة الصوت واستخراج الكلمات المكتوبة وتحديد توقيتاتها بدقة.
* **YouTube Service (`youtube.py`):** البحث عن مقاطع فيديو ذات صلة وتحميلها لاستخدامها كمشاهد خلفية.

## 3. بنية واجهة المستخدم (Frontend Architecture)

تم بناء الواجهة باستخدام **Next.js 15 (App Router)** مع **Tailwind CSS** للتصميم الجمالي الفاخر.

### أ. الصفحات والمسارات (App Routes)
1. **الرئيسية (`/`):** تحتوي على الـ Hero section ومدخل المشروع الأساسي (تحويل فكرة لفيديو).
2. **لوحة التحكم (`/dashboard`):** عرض المشاريع السابقة وحالتها.
3. **التحليل (`/analyze`):** لوحة تعرض السكريبت المولد للمشاهد والمقاطع المقترحة قبل الرندر.
4. **الاستوديو (`/studio`):** المحرر الرئيسي للفيديو، يتيح تغيير المقاطع، النسبة (16:9, 9:16)، وتحديد الصوت، وتشغيل رندر الفيديو النهائي وتحميله.

### ب. المكونات والتفاعل (Components & API Link)
* **[HeroInput.tsx](file:///c:/Users/youse/OneDrive/Desktop/يوتيوب/frontend/components/HeroInput.tsx):** حقل الإدخال الذكي في الصفحة الرئيسية المعتمد على تفاعل مميز لدعم تجربة مستخدم مبهرة.
* **[api.ts](file:///c:/Users/youse/OneDrive/Desktop/يوتيوب/frontend/lib/api.ts):** يمثل صلة الوصل بين الـ Frontend وسيرفر الـ Backend لتنفيذ كل الطلبات بسلاسة.
